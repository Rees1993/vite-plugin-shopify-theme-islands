## Agent Brief

**Category:** enhancement  
**Summary:** Preserve explicit `tagSource: "filename"` as the compatibility mode for upgrades while aligning diagnostics and migration guidance with the v2.0 default.

**Current behavior:**  
Filename ownership is the only mode. Diagnostics and docs assume the filename is the source of Tag ownership.

**Desired behavior:**  
After the new default lands, consumers should still be able to opt into the prior filename-based ownership model explicitly with `tagSource: "filename"`. In that mode, ownership behavior should remain consistent with pre-v2.0 expectations, and diagnostics should continue to treat filename-vs-Registered-Tag mismatches as useful warnings. Public docs and migration notes should explain that `registeredTag` is the v2.0 default while `filename` remains available for compatibility.

**Key interfaces:**  
- Plugin config shape — explicit `tagSource: "filename"` should preserve the legacy ownership model  
- Diagnostics policy — filename-vs-Registered-Tag mismatch warnings should only apply in filename mode  
- Public docs and migration guidance — should explain the new default, the compatibility mode, and when to choose each

- [ ] `tagSource: "filename"` preserves the existing filename-based Tag ownership model
